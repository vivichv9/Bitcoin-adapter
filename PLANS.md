# План разработки Bitcoin Adapter

## Этап 1. Инициализация проекта
1. Создать структуру веб-приложения на JavaScript.
2. Настроить базовый UI-каркас (форма, секции результатов, блоки валидации).
3. Подключить инструменты качества: линтер, форматтер, базовые тесты.
4. Описать конфигурации окружений (dev/test/prod).

## Этап 2. Модуль форм и валидации
1. Спроектировать JSON-схемы для основных пользовательских форм.
2. Реализовать клиентскую валидацию обязательных полей и типов.
3. Добавить подсказки по форматам ввода (hex, pubkey, txid, amount).
4. Сделать отображение ошибок в понятном виде.

## Этап 3. Конвертация в hex
1. Реализовать слой нормализации входных данных.
2. Реализовать сериализацию в формат, совместимый с Bitcoin RPC.
3. Добавить превью итогового hex с возможностью копирования.
4. Покрыть модуль unit-тестами на типовые и ошибочные кейсы.

## Этап 4. Интеграция с Bitcoin RPC
1. Добавить модуль формирования payload для `generateblock`.
2. Добавить модуль формирования payload/raw hex для `sendrawtransaction`.
3. Реализовать обработку RPC-ошибок и понятные сообщения пользователю.
4. Подготовить слой конфигурации узла (host, port, auth, network).

## Этап 5. Генерация адресов
1. Реализовать генерацию адресов из приватного/публичного ключа.
2. Поддержать базовые типы адресов (P2PKH, P2SH, Bech32 и др.).
3. Добавить проверку корректности полученного адреса.
4. Добавить тесты совместимости с известными тест-векторами.

## Этап 6. Проверка подписи транзакций
1. Реализовать форму ввода данных для проверки подписи.
2. Добавить модуль проверки подписи для поддерживаемых типов входов.
3. Выводить результат: `valid/invalid` + причина при ошибке.
4. Добавить набор проверок на граничные случаи.

## Этап 7. Скрипты по публичному ключу
1. Реализовать анализ публичного ключа.
2. Генерировать перечень всех поддерживаемых script-типов.
3. Для каждого script-типа показывать шаблон и итоговый scriptPubKey.
4. Добавить экспорт результатов (JSON/текст).

## Этап 8. UX и эксплуатация
1. Добавить историю операций (локально в браузере).
2. Реализовать удобное копирование/сохранение результатов.
3. Добавить базовые меры безопасности (маскирование чувствительных данных).
4. Подготовить документацию пользователя и разработчика.

## Этап 9. Тестирование и релиз
1. Провести интеграционные тесты с локальным Bitcoin Core node.
2. Выполнить регрессионное тестирование ключевых сценариев.
3. Подготовить релиз-кандидат и changelog.
4. Выпустить v1 и собрать обратную связь для v1.1.
