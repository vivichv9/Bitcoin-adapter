# Документация изменений от 2026-02-11 (этап 8: UX и эксплуатация)

## Что реализовано
1. История операций в браузере:
- добавлен модуль `src/lib/history.js`;
- операции сохраняются в `localStorage`;
- хранится ограниченное число последних записей.

2. UI для истории:
- в `index.html` добавлена секция "История операций";
- кнопки:
  - экспорт истории в JSON;
  - очистка истории.

3. Интеграция истории в клиентскую логику:
- `src/main.js` теперь пишет в историю ключевые действия:
  - генерация hex;
  - RPC вызовы;
  - генерация/валидация адресов;
  - проверка подписи;
  - экспорт script-типов.

4. Базовые меры безопасности:
- RPC пароль больше не сохраняется в `localStorage` при сохранении node config;
- история редактирует чувствительные поля (`password`, `token`, `secret` -> `<redacted>`);
- длинные hex-значения в истории маскируются.

5. Тесты:
- добавлен `tests/history.test.mjs`;
- обновлены `scripts/lint.mjs` и `scripts/build.mjs` для нового модуля/теста.

6. README:
- добавлен раздел про UX, историю и безопасность.

## Результат
Пользователь получил удобную эксплуатационную функцию истории действий с безопасным хранением локальных данных и быстрым экспортом для анализа.
